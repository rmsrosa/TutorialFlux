Identificação de dígitos com os dados MNIST¶

  • A base de dados MNIST um subconjunto de um conjunto de dgitos escritos mo disponibilizados pelo National Institute of Standards and Technology (NIST), do Departamento de Comrcio Americano.

  • A base est disponvel em JuliaML/MLDatasets.jl.

  • Veja mais sobre o MLDatasets: The MNIST database of handwritten digits.

  • O banco de dados contm 60 mil imagens para treino e 10 mil para teste.

  • Cada imagem em resoluo de 28x28 pixels de nveis inteiros de cinza entre 0 e 255.

Rede¶

  • Como cada pixel um nvel de cinza entre 0 e 1, vamos "aumentar o contraste" com uma camada inicial da forma $$ x -> \sigma(100(x-0.5)) = \frac{e^{100(x-0.5)}}{1 + e^{100(x-0.5)}} $$

Função de custo, otimizador e callback¶

  • Aqui, usamos logitcrossentropy como funo de custo.

  • logitcrossentropy equivalente ao crossentropy mas mais estvel numericamente.

  • Ambos so muito usados em problemas de classificao, especialment em conjunto com uma camada final softmax e com os classificaes dadas em fomat one-hot/one-cold.

  • O bom ADAM como otimizador.

  • E uma funo de callback para acompanharmos o treinamento.

In [11]:
loss(x, y) = Flux.Losses.logitcrossentropy(m(x), y)

opt = ADAM();

function cb(i, shown; showstep = 1000) # funo de call-back, executada a cada iterao
    global shown
    if i > shown
        @show i, loss(data_x,data_y)
        shown += showstep
    end
    return nothing
end

Treinamento¶

  • Vamos fazer uma otimizao estocstica em lotes.

  • A ideia dividir os dados em quatro blocos.

  • Vamos usar o DataLoader para facilitar o processo de dividir os dados em lotes.

In [12]:
data_batch = Flux.Data.DataLoader((data_x, data_y), batchsize=div(last(size(data_y)),4))

n_epochs = 1_000
shown = 0
istep = 100
@time for i in chain(1:istep-1:istep, 2*istep:istep:n_epochs) # frescura
    @time Flux.train!(loss, params(m), ncycle(data_batch, istep), opt, cb = () -> cb(i, shown, showstep=istep))
    println()
end

Testando o resultado do treinamento¶

  • Agora, vamos testar a rede treinada usando os dados de testetest_x e test_y.

  • Com o softmax, que usado em problemas de classificao, a sada da rede um vetor indicando a probabilidade da imagem ser um ou outro dgito.

  • Comparamos, ento, o dgito com a maior probabilidade com o dgito do teste.

  • Podemos fazer isso vendo o ndice do elmento de maior valor, com argmax, ou usar a funo Flux.onecold, que, nesse caso, retorna o ndice com a maior probabilidade. Como o primeiro ndice corresponde ao dgito zero, devemos subtrair 1 para termos o dgito correspondente.

  • A acurcia a porcentagem de acertos.

Outros treinos¶

  • Uma sugesto que refaam o treinamento com outras redes (mais neurnios em uma camada, mais camadas, com camadas convolucionais).

  • Tambm com mais e menos dados do que os utilizados (lembrem que usei, acima, apenas parte das 60 mil imagens).

  • E com outros tamanhos de lote.

  • E que comparem a acurcia e o tempo de treinamento com a rede acima.

  • Nessa comparao, lembrem-se que a inicializao dos parmetros da rede aleatria e que isso influencia na convergncia do treinamento. Portanto, essa comparao s tem validade cientfica, mesmo, com muitos treinamentos.
